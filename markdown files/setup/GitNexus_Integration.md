# GitNexus Integration Guide (Gem)

This document provides a step-by-step guide for integrating **GitNexus** into your professional development workflow involving **Gemini CLI**, **Copilot CLI**, and **Warp**.

## Workflow Architecture
| Tool | Role | Integration |
| :--- | :--- | :--- |
| **GitNexus** | The Knowledge Graph | Indexes code structure and dependencies. |
| **Gemini CLI** | The Architect | Uses GitNexus documentation to perform deep, safe edits. |
| **Copilot CLI** | The Assistant | Provides quick terminal command suggestions for GitNexus. |
| **Warp** | The Command Center | Runs analysis and visualizes the knowledge graph. |

---

## Setup & Configuration

### 1. Provider Selection
When prompted by GitNexus, choose **OpenRouter** or **OpenAI**. 
*   **OpenRouter (Recommended)**: Best for Gemini users. It allows you to use `google/gemini-pro-1.5` through a single API key.
*   **Custom Endpoint**: Avoid unless running local proxies (like LiteLLM or Ollama).

### 2. Initialization
Run these commands in a **Warp** terminal to build your project's index:

```bash
# Build the local knowledge graph (No LLM cost)
npx gitnexus analyze

# Generate agent-ready documentation (Uses LLM provider)
npx gitnexus wiki
```

## Daily Workflow

### Phase 1: Knowledge Enrichment (Warp)
Whenever you make significant architectural changes (e.g., updating `db/schema.ts` or adding complex components), refresh the index:
1.  Open a tab in Warp.
2.  Run `npx gitnexus analyze`.
3.  (Optional) Run `npx gitnexus serve` to explore the "Blast Radius" of your changes in the web UI.

### Phase 2: Implementation (Gemini CLI)
Before starting a task that requires codebase understanding, ensure Gemini CLI has the latest context:
*   **Instruction**: "Gemini, read the `AGENTS.md` and `CLAUDE.md` files generated by GitNexus."
*   **Result**: Gemini will now understand the relationship between your Drizzle tables, Next.js routes, and UI components without "blind edits."

### Phase 3: Command Help (Copilot CLI)
If you forget GitNexus flags or commands:
*   **Prompt**: `gh copilot suggest "gitnexus command to check index freshness"`
*   **Result**: It will suggest `npx gitnexus status`.

---

## Best Practices
*   **Commit Documentation**: GitNexus creates/updates `AGENTS.md` and `CLAUDE.md`. Keep these in your repo so they serve as "Source of Truth" for all AI agents.
*   **Update Frequently**: An outdated index is dangerous. Run `analyze` after every major PR or schema change.
*   **Use the UI**: The `npx gitnexus serve` command provides a visual graph that is excellent for debugging complex circular dependencies.
